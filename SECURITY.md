# Security Policy

## Supported Versions

We release patches for security vulnerabilities for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability within openai-structured, please send an email to <yaniv@golan.name>. All security vulnerabilities will be promptly addressed.

Please include the following information:

- Type of issue (e.g. buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

## Security Updates

Security updates will be released as soon as possible after they are developed and tested. We will notify users through:

- GitHub Security Advisories
- Release notes
- Our documentation

## Best Practices

When using openai-structured:

1. Never commit API keys or other secrets to version control
2. Use environment variables for sensitive configuration
3. Keep the library updated to the latest version
4. Follow secure coding practices when implementing custom handlers

## Contact

If you have any questions about this security policy, please email <yaniv@golan.name>.
